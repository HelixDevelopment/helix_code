## Recording Requirements

### Software
- Screen recorder (OBS Studio or Camtasia)
- Video editor (DaVinci Resolve or Premiere)
- Audio editor (Audacity)
- VS Code with Go extension
- Terminal emulator
- Browser for API testing

### Hardware
- Microphone (USB condenser recommended)
- Pop filter
- Quiet recording environment
- 1080p or 4K display
- Sufficient RAM for smooth recording

### Recording Settings
- Resolution: 1920x1080 (minimum)
- Frame rate: 30fps
- Audio: 48kHz, 16-bit
- Format: MP4 (H.264)

## Video Structure Template

Each video follows this structure:

```
[0:00] - Title card with video name
[0:10] - Introduction and objectives
[0:30] - Content section 1
[X:XX] - Content section 2
[X:XX] - Content section 3
[X:XX] - Live demo
[X:XX] - Key takeaways summary
[X:XX] - Next video preview
[END]  - End card with resources
```

## Editing Guidelines

### Visual Elements
- Consistent intro/outro animations
- Chapter markers for navigation
- Code highlighting overlays
- Zoom effects for important areas
- Picture-in-picture for demos

### Audio
- Background music (subtle, instrumental)
- Sound effects for transitions
- Normalized audio levels (-16 LUFS)
- Noise reduction applied

### Text Overlays
- Key points as bullet overlays
- Code snippets as full-screen
- Error messages highlighted
- URLs and resources shown
