Production hang found + fixed

The streaming e2e exposed a real production defect (HXC-104): streamLLM’s goroutine called provider.GenerateStream(ctx, llmReq, chunkChan) but never closed chunkChan, so the SSE consumer never observed channel-drain, never emitted [DONE], and every /stream request hung until the 120s context deadline. Fix: defer close(chunkChan) in that goroutine (internal/server/llm_generate.go). This is exactly the class of user-facing breakage that handler/httptest tests miss and a real runtime e2e catches.
