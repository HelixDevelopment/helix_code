The 401/200 auth matrix was therefore proven against (a) an ephemeral key-checking server on :18439 and (b) a real httptest server in the integration test; both mirror the llama.cpp --api-key Bearer contract. The provider’s own header emission (Authorization: Bearer <key>) is proven real.

The client composition sits BELOW the endpoint seams in precedence, so a default deployment (HELIX_LLM_ENDPOINT set) is unaffected; a leaked bind-all 0.0.0.0 maps to localhost.
