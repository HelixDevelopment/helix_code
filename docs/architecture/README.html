Honesty note (§11.4 / §11.4.123)

Both documents describe the target architecture. They explicitly separate IMPLEMENTED components (real code, cited file:line) from PLANNED components (specified in SP1/SP2/SP4 but not yet built — no captured runtime evidence). A PLANNED node is a design intent, not a claim of working software. The runtime-evidence bar (§11.4.5 / §11.4.69 / §11.4.107) is met only by the tests/Challenges each SP plan requires at execution time — not by these diagrams or this DDL.
